🌐 Deployment map — where each piece runs (and why serverless only goes so far)

Frontend (apps/web)→ Netlify — static, free, serverless ✓ = solo practice, the public demo, works today
Game server (Phase 2)→ always-on Node process — NOT serverless holds live 60-tick rooms over WebSockets · your Pentium box* or a $5/mo VPS
Data / auth / boards→ Supabase — managed, serverless ✓ anon accounts, match history, daily/global leaderboards

* Your Pentium dual-core / 4GB DDR2 can host the room server for a small community (the workload is tiny) — if its CPU is new enough to run modern Node (test: install Node, run node -v; an "illegal instruction" crash = CPU too old, use a VPS instead). It must also be always-on and internet-reachable (public IP + port forward, or a Cloudflare Tunnel).
Fully-serverless multiplayer is possible only by re-architecting the room server onto Cloudflare Durable Objects / PartyKit — a deliberate departure from the documented Node + Socket.IO design, so it's an ADR decision, not a default.

SOON To be solid

Phase 1 closeout, ~3–4 weeks: playtest and tune the feel (the game's whole thesis), the performance-hygiene fixes from the review, cross-engine determinism CI, and the documentation consolidation.

BIG To be the vision

Phase 2 is the real lift, ~10–12 weeks: the authoritative game server, the netcode, Supabase, and the anti-cheat validator — this is where "up to 4 players, same world, live board" becomes real.

Estimates assume one developer. A part-time solo maintainer (the vision's realistic scenario) should roughly double the calendar. The critical path to a public playable demo is short — the Release Prep block. The critical path to the multiplayer vision runs through Phase 2, which is genuinely the largest chunk of remaining work. Everything before Phase 2 is already built and tested.
